# ESP32 裁判计分客户端

这是比赛计分系统的裁判手持端，对应主机项目 [score_system](../score_system)。

每个裁判使用 1 块 ESP32、5 个按键和 1 个 TM1637 数码管：
- 红方 `+1`
- 红方 `+2`
- 蓝方 `+1`
- 蓝方 `+2`
- 提交键

裁判在本机录入本轮分数，按提交键后通过 WiFi 发送到主机。

## 当前规则

- 每轮每个裁判只能提交 1 次
- 提交成功后，本裁判客户端立即锁定
- 锁定时数码管显示 `----`
- 只有网页端点击“下一轮”后，客户端才会解锁并恢复到 `00:00`
- 如果提交失败，当前分数不会丢失，可以重试

## 硬件

### 连接方式

按钮一端接 GPIO，另一端接 GND，代码使用 `INPUT_PULLUP`。

| 功能 | GPIO |
|---|---|
| 红方 +1 | `GPIO12` |
| 红方 +2 | `GPIO27` |
| 蓝方 +1 | `GPIO14` |
| 蓝方 +2 | `GPIO26` |
| 提交键 | `GPIO13` |

TM1637：

| TM1637 引脚 | GPIO |
|---|---|
| CLK | `GPIO2` |
| DIO | `GPIO15` |
| VCC | `3.3V` 或 `5V` |
| GND | `GND` |

## 使用流程

1. 先启动主机 `score_system`
2. 再给客户端上电
3. WiFi 未连接时数码管显示 `FFFF`
4. 连接主机后显示当前可录入比分，例如 `00:00`
5. 裁判录入本轮分数后按提交键
6. 提交成功后数码管显示 `----`
7. 等网页点击“下一轮”后，客户端恢复 `00:00`

## 显示含义

| 显示 | 含义 |
|---|---|
| `FFFF` | WiFi 未连接主机 |
| `00:00` ~ `99:99` | 当前本轮分数 |
| `----` | 本轮已提交，等待网页开启下一轮 |

## 故障排查

| 现象 | 说明 |
|---|---|
| 一直显示 `FFFF` | 主机未启动，或 WiFi 未连上 |
| 按提交后分数没清零且闪烁 | 提交失败，分数保留，可再次提交 |
| 提交后显示 `----` | 正常，表示本轮已提交完成 |
| 一直停在 `----` | 需要网页端点击“下一轮” |
