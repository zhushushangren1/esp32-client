# ESP32 裁判计分客户端

武术 / 散打类比赛电子计分系统的**裁判手持端**。每位裁判持一台，通过 5 个按钮记录红/蓝双方得分，按绿色提交键后通过 WiFi 把本轮分数上报给主机 `score_system`。

本项目为 [`score_system`](../score_system) 的配套客户端，**必须与主机配合使用**。完整系统需烧录 **3 台** ESP32 作为 3 位裁判的手持设备。

---

## 功能特性

- 5 键记分：红方 +1 / +2、蓝方 +1 / +2、绿色提交键
- TM1637 四位数码管实时显示分数（格式：`红红:蓝蓝`，中间带冒号）
- WiFi 常驻连接 + 断线自动重连，上报延迟 < 200 ms（旧版本 3~15 秒）
- **未就绪提示**：WiFi 未连上时数码管显示 `FFFF`，连上后自动切换为分数
- **上报失败保护**：提交失败不清零分数、数码管闪两下提示重试
- 按钮启动时自读状态防上电误触发，50 ms 消抖

## 接线

### 按钮（一端接 GPIO，另一端接 GND；代码里已启用内部上拉电阻）

| 功能 | ESP32 引脚 | 丝印 |
|---|---|---|
| 红方 +1 | GPIO 12 | D12 |
| 红方 +2 | GPIO 27 | D27 |
| 蓝方 +1 | GPIO 14 | D14 |
| 蓝方 +2 | GPIO 26 | D26 |
| 提交 / 清零（绿色）| GPIO 13 | D13 |

### TM1637 数码管

| TM1637 引脚 | ESP32 引脚 |
|---|---|
| VCC | 3.3V 或 5V |
| GND | GND |
| CLK | GPIO 2 (D2) |
| DIO | GPIO 15 (D15) |

### 接线示意

```
       ESP32                       TM1637
   ┌───────────┐               ┌────────┐
   │   GPIO12  │←── 红+1       │        │
   │   GPIO27  │←── 红+2       │        │
   │   GPIO14  │←── 蓝+1       │        │
   │   GPIO26  │←── 蓝+2       │        │
   │   GPIO13  │←── 绿键        │        │
   │       GND ├── 所有按钮另一端  │       │
   │           │               │        │
   │      3.3V ├───────────────┤ VCC    │
   │       GND ├───────────────┤ GND    │
   │    GPIO2  ├───────────────┤ CLK    │
   │   GPIO15  ├───────────────┤ DIO    │
   └───────────┘               └────────┘
```

## 烧录步骤

### 前置条件

- 已安装 VSCode + PlatformIO 插件
- 已准备好 3 台 ESP32（硬件烧录完全相同，只有源码中 `CLIENT_ID` 不同）

### 1. 打开项目

VSCode → `File → Open Folder` → 选 `esp32-client`

### 2. 修改客户端 ID

每台设备需要**分别修改**源码后再烧录。打开 `src/main.cpp`，第 7 行：

```cpp
const char* CLIENT_ID = "client2";  // 根据实际设备修改此值
```

三台设备分别设置为 `"client1"` / `"client2"` / `"client3"`，**大小写敏感**。

### 3. 编译烧录

- VSCode 左下状态栏点 `→` 图标（Upload）
- 或 PlatformIO 侧边栏：PROJECT TASKS → esp32dev → General → **Upload**

重复 3 次：每次改 `CLIENT_ID`、换一台设备、烧录。

## 使用方式

### 启动顺序

1. **先开主机** `score_system`（等它创建 `ScoreSystem` 热点）
2. **再开本客户端**
3. 数码管先显示 `FFFF`（WiFi 未就绪），1~3 秒后自动变为 `00:00` 表示已连上主机

### 比赛操作流程

1. 双方得分时按对应颜色 +1 / +2 键 → 数码管实时显示当前本轮分数
2. 一轮结束后按**绿色**提交键：
   - **成功**：数码管清零为 `00:00`
   - **失败**：闪两下后保留原分数，等 WiFi 恢复再按一次
3. 三位裁判都提交完本轮后，主机自动按 2/3 多数规则裁决并累加总分

### 分数范围

0 ~ 99。超过 99 后数码管截断显示为 99，但内部继续累加，提交时发送真实值。

## 故障排查

| 现象 | 可能原因与处理 |
|---|---|
| 数码管长期 `FFFF` | 主机没开 / WiFi 密码错 / 距离太远。检查 `score_system` 是否先启动 |
| 按绿键后闪两下没清零 | 提交失败。等 WiFi 重连后重试（数码管变 `00:00` 前会一直失败）|
| 按钮无响应或自动跳分 | 按钮虚焊 / 上下电瞬间尖峰。先断电重启试试 |
| 串口看不到 `WiFi connected` | 主机 AP 没起来，或本机和主机不在 2.4GHz 范围 |
